                nftables Rule Syntax Guide
                

                    

                        This platform uses nftables (Linux kernel packet filter) instead of traditional Cisco-style ACLs.
                        Each rule is a single nftables match/action fragment. The system automatically wraps it with the correct table, chain, interface binding, and comment tag.
                    

                    
Common Rule Examples

                    

                        
                            
RuleWhat It Does

                        
                        
                            
tcp dport 22 acceptAllow SSH

                            
tcp dport {80, 443} acceptAllow HTTP and HTTPS

                            
udp dport 53 acceptAllow DNS

                            
icmp type echo-request acceptAllow ping

                            
ip saddr 10.10.1.0/24 acceptAllow traffic from a subnet

                            
ip saddr 192.168.1.100 dropBlock a specific host

                            
tcp dport 22 ip saddr 10.10.1.0/24 acceptAllow SSH from a subnet only

                            
dropDrop everything else (default deny)

                        
                    


                    
Key Differences from Cisco ACLs

                    
    
                        
  • accept / drop instead of permit / deny
    • 
                        
  • Protocol comes before the port: tcp dport 80 not port 80 tcp
    • 
                        
  • Use curly braces for multiple ports: tcp dport {22, 80, 443}
    • 
                        
  • CIDR notation for subnets: ip saddr 10.0.0.0/8
    • 
                        
  • Rules are evaluated top-to-bottom — put drop last for default deny
    • 
                    


                    
How It Works Internally

                    

                        Rules are applied via an Ansible playbook that runs nft commands inside the device container.
                        Each ACL is tagged with a comment (e.g., nm_acl_LAB-IN) for idempotent removal.
                        Rules are added to the inet nm_acl table under an input or output chain depending on direction.
